Introduction

 

A well-lit figurine is placed cm in front of a converging lens with a focal length of 15 cm. A clear image is focused on a screen. What is the magnification of the image?

What number should I put in the blank to make a good test question?

 

Lens Problem Designer is aimed at physics instructors who are writing homework, quiz, and test questions that involve the thin lens equation. It can also be used to solve such problems and explore what-if scenarios.

 

There are two ways to experiment with values and solutions. In the base mode simply type in two values and the program calculates the results. You can enter different values and see how the numbers change. In the list-generating mode you set up a range of values for one of the quantities and a list of solutions is placed in the List Area. See List Generation.

 

Philosophy

I like to write problems that have “nice” answers. By “nice” I mean that when you assume numbers in the problem are good to three significant digits and work the problem using at least four sig figs, the answer can be written with only two sig figs but be good to three. Suppose the given values are 4 and 18 and the answer is 18 divided by 4. The answer of 4.5 is “nice” but division by four (or 5, or 10) gets old fast. A more interesting value would be 5.3 instead of 4. Now when you divide you get 3.3962264, which when rounded to three sig figs would be 3.40. If we are assuming each number is good to 3 sig figs, then the result can be written as just 3.4. This nice answer comes from values that look like they could be from a randomizer. It's an eccentricity, I'll admit; but I like it. This program was written to make this task easier for thin lens problems (and to learn how to program in Python).

Operation

There are two equations and four variables, so two quantities must be known and two unknown quantities can be found. In Lens Problem Designer a checkbox is marked next to two values to indicate that they are the known values and numbers are then placed in the entry boxes. As numbers are entered, the program automatically calculates the values of the unknowns. It is also possible to set one of the known values to a range and create a list of solutions for that range. See List Generation.

The Interface

 

Exactly two check boxes must be checked for the program to calculate values. Results are updated as the user types numbers.

 

List Generation

In list generation mode a variable will sweep through a range and the results will be in the List Area. For list generation the radio button for the ranging quantity is selected. It must also have a check in its checkbox. Since this program was written as an exercise in learning Python, the range is entered in a Python format. The format is start:end<:step = 1> where the first value used will be start, and the last value will be less than end. If step is entered then the value goes up in increments of step. If not specified the step will be one. For example 1:5 will evaluate solutions with the variable set to 1, 2, 3, and 4, not including 5. I chose that interpretation because Python does it. Sometimes the end value will show up due to the way that floating point decimal numbers are handled by computers. Call it a feature, a bonus value in the list, and not a bug. The range 2.5:6:0.5 evaluates for 2.5, 3, 3.5, 4, 4.5, 5, and 5.5. You can also separate the numbers with commas, but they will be replaced with colons on the screen. To generate the list, click on the List Area or select Generate List from the List menu. There's a demo of a list available from the List menu.

Menus

The List menu contains three options. Generate List does just that. Clear List deletes all entries from the List Area like the button with the same name. List Demo enters a focal length and a range for the object distance and generates a list. In the list generated there are three “nice” magnifications. When written to 3 sig figs and rounded, they only have two digits. Change the sig figs to 4 or more and generate another list to see the longer results that got rounded. These values of the object distance would be great choices to put in the problem stated in the introduction.

 

 

The Reset menu items are straightforward. Reset Entries sets the entries, checkboxes, and radio buttons to their initial state. Clear List does just that. Reset All performs the functions of both other menu items.
